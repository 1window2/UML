### Use Case: 회원가입
* **Actor:** 비회원
* **Description:** 시스템을 이용하기 위해 새로운 계정을 생성한다.
* **Precondition:** 시스템에 접속한 비로그인 상태여야 한다.
* **Postcondition:** 필수 입력 정보가 시스템에 저장되고, 로그인할 수 있는 권한을 얻는다. 
* **Main Flow:**
  1. 비회원이 회원가입을 요청한다.
  2. 시스템은 회원가입 폼을 출력한다.
  3. 비회원은 필수 정보(ID, 비밀번호, 이름, 전화번호, 이메일)를 입력하고 가입을 완료한다. (Req. 1)
  4. 시스템은 회원 정보를 등록한다.

### Use Case: 로그인
* **Actor:** 회원, 관리자
* **Description:** 서비스 이용을 위해 사용자 인증을 거친다.
* **Precondition:** 시스템에 계정이 등록되어 있어야 한다.
* **Postcondition:** 인증에 성공하여 권한에 맞는 기능을 사용할 수 있게 된다. 
* **Main Flow:**
  1. Actor가 ID와 비밀번호를 입력하여 로그인을 요청한다.
  2. 시스템은 입력 정보를 확인하여 인증을 완료한다. (단, 관리자는 주어진 ID `admin`, 비밀번호 `admin`을 사용한다.) (Req. 3)

### Use Case: 로그아웃
* **Actor:** 회원, 관리자
* **Description:** 시스템 이용을 마치고 현재 세션을 종료한다.
* **Precondition:** 로그인되어 있는 상태여야 한다.
* **Postcondition:** 세션이 종료되고 비로그인 상태로 전환된다. 
* **Main Flow:**
  1. Actor가 언제든 로그아웃을 요청한다.
  2. 시스템은 로그인 세션을 종료하고 초기 화면으로 이동한다. (Req. 4)

### Use Case: 회원 탈퇴
* **Actor:** 회원
* **Description:** 회원이 서비스 이용을 중단하고 자신의 데이터를 삭제한다.
* **Precondition:** 회원으로 로그인되어 있어야 한다.
* **Postcondition:** 회원의 이용 권한과 관련 데이터가 영구 삭제된다. 
* **Main Flow:**
  1. 회원이 언제든 회원 탈퇴를 요청한다.
  2. 시스템은 해당 회원의 모든 이용 권한과 데이터를 삭제 처리하고 비로그인(초기) 화면으로 이동한다. (Req. 2)

### Use Case: 설문 검색
* **Actor:** 회원
* **Description:** 조건과 키워드를 통해 참여할 설문을 찾는다.
* **Precondition:** 회원으로 로그인되어 있어야 한다.
* **Postcondition:** 조건에 맞는 설문 리스트가 출력된다. 
* **Main Flow:**
  1. 회원이 설문 리스트(현재 진행 중, 향후 진행 예정, 종료된 설문 중 하나)를 선택한다. (Req. 9)
  2. 회원이 검색 키워드를 입력하여 검색을 요청한다.
  3. 시스템은 검색 결과로 조건에 맞는 설문 리스트를 출력한다. (Req. 10)

### Use Case: 상세정보 조회 [회원]
* **Actor:** 회원
* **Description:** 검색된 특정 설문의 세부 정보와 문항을 확인한다.
* **Precondition:** [설문 검색] 결과 리스트가 출력된 상태여야 한다.
* **Postcondition:** 선택한 설문의 상세 정보가 새로운 화면에 표시된다. 
* **Main Flow:**
  1. 회원이 검색된 설문 리스트에서 특정 설문을 선택한다.
  2. 시스템은 새로운 화면에 설문 제목, 설문 설명, 설문 문항, 응답 항목, 설문 시작 시각, 설문 마감 시각 등을 출력한다. (Req. 11, 12)
  3. 시스템은 선택한 설문이 현재 진행 중인 경우, 응답 버튼을 활성화하여 출력한다. (Req. 13)

### Use Case: 설문 응답
* **Actor:** 회원
* **Description:** 진행 중인 설문에 참여하여 응답을 제출한다.
* **Precondition:** [상세정보 조회 [회원]] 화면에서 응답 버튼이 활성화되어 있어야 한다.
* **Postcondition:** 회원의 응답 정보가 시스템에 저장된다. 
* **Main Flow:**
  1. 회원이 활성화된 응답 버튼을 누른다.
  2. 시스템은 응답 화면으로 이동시킨다.
  3. 회원이 설문에 응답하고 응답(제출) 버튼을 눌러 제출을 완료한다. (Req. 14)

### Use Case: 응답 설문 조회
* **Actor:** 회원
* **Description:** 자신이 지금까지 응답한 설문 기록을 확인한다.
* **Precondition:** 회원으로 로그인되어 있어야 한다.
* **Postcondition:** 회원이 응답한 모든 설문 정보가 한 화면에 출력된다. 
* **Main Flow:**
  1. 회원이 응답 설문 조회를 요청한다.
  2. 시스템은 회원이 응답한 모든 설문 정보를 한 화면에서 출력한다. (Req. 15)
  3. 시스템은 각 응답한 설문에 대해 설문 제목, 응답 내용, 응답 일시 등을 출력한다. (Req. 16)

### Use Case: 응답 수정
* **Actor:** 회원
* **Description:** 자신이 이전에 제출했던 설문 응답을 수정한다.
* **Precondition:** 회원으로 로그인되어 있고, 자신이 참여한 설문이어야 한다.
* **Postcondition:** 기존 응답 데이터가 수정된 데이터로 갱신된다. 
* **Main Flow:**
  1. 회원이 자신이 참여한 '현재 진행 중'인 설문에 대해 수정을 요청한다.
  2. 시스템은 설문 수정 화면을 출력하며, 화면에 가장 최근에 수정한 날짜를 함께 출력한다. (Req. 18)
  3. 회원이 응답을 수정하고 수정 버튼을 눌러 완료한다. (Req. 17) 
* **Exception Flow (예외 처리):**
  * 1a. 설문이 이미 종료된 경우: 시스템은 응답 내용을 수정할 수 없도록 제한한다. (Req. 20)

### Use Case: 응답 취소
* **Actor:** 회원
* **Description:** 자신이 제출했던 설문 응답을 철회한다.
* **Precondition:** [응답 설문 조회]를 통해 응답 내역이 확인된 상태여야 한다.
* **Postcondition:** 회원의 해당 설문 응답 데이터가 삭제된다. 
* **Main Flow:**
  1. 회원이 [응답 설문 조회] 목록에서 '현재 진행 중'인 설문에 대해 취소를 요청한다.
  2. 시스템은 해당 설문 응답 내역을 삭제(취소) 처리한다. (Req. 19) 
* **Exception Flow (예외 처리):**
  * 1a. 설문이 이미 종료된 경우: 시스템은 응답 내용을 취소할 수 없도록 제한한다. (Req. 20)

### Use Case: 설문 등록
* **Actor:** 관리자
* **Description:** 사용자들에게 제공할 새로운 설문을 등록한다.
* **Precondition:** 관리자로 로그인되어 있어야 한다.
* **Postcondition:** 새로운 설문이 시스템에 등록된다. 
* **Main Flow:**
  1. 관리자가 설문 등록을 요청한다.
  2. 시스템은 설문 등록 화면을 출력한다.
  3. 관리자는 설문 제목, 설문 설명, 설문 문항, 응답 항목, 설문 시작 시각, 설문 마감 시각 등을 입력하여 등록을 완료한다. (Req. 5)

### Use Case: 설문 조회
* **Actor:** 관리자
* **Description:** 시스템에 등록된 전체 설문 목록을 확인한다.
* **Precondition:** 관리자로 로그인되어 있어야 한다.
* **Postcondition:** 전체 설문 목록이 화면에 출력된다. 
* **Main Flow:**
  1. 관리자가 설문 관리를 요청한다.
  2. 시스템은 등록된 모든 설문 리스트를 화면에 출력한다. (Req. 6)

### Use Case: 상세정보 조회 [관리자]
* **Actor:** 관리자
* **Description:** 등록된 특정 설문의 세부 내용을 팝업 창으로 확인한다.
* **Precondition:** [설문 조회] 리스트가 화면에 출력된 상태여야 한다.
* **Postcondition:** 선택한 설문의 상세 내용이 팝업으로 출력된다. 
* **Main Flow:**
  1. 관리자가 설문 리스트 조회 화면에서 특정 설문 항목을 선택한다.
  2. 시스템은 해당 설문의 상세 내용을 팝업 창으로 출력하여 보여준다. (Req. 7)

### Use Case: 설문 삭제
* **Actor:** 관리자
* **Description:** 시스템에 등록된 특정 설문을 삭제한다.
* **Precondition:** [설문 조회] 리스트가 화면에 출력된 상태여야 한다.
* **Postcondition:** 선택한 설문이 시스템에서 영구 삭제된다. 
* **Main Flow:**
  1. 관리자가 설문 리스트 조회 화면에서 특정 설문 항목을 선택하여 삭제를 요청한다.
  2. 시스템은 해당 설문을 목록과 데이터베이스에서 삭제 처리한다. (Req. 8)

### Use Case: 설문 통계 조회
* **Actor:** 관리자
* **Description:** 지정한 기간 동안의 설문 운영 및 응답 통계 지표를 확인한다.
* **Precondition:** 관리자로 로그인되어 있어야 한다.
* **Postcondition:** 조건에 맞는 통계 수치가 화면에 출력된다. 
* **Main Flow:**
  1. 관리자가 통계 조회를 요청한다.
  2. 시스템은 통계 조회 화면을 출력한다.
  3. 관리자가 검색 단위(최근 1주일, 1개월, 1년 중 선택)를 선택한다. (Req. 21)
  4. 시스템은 선택한 기간 동안의 '전체 설문 수', '종료된 설문 수', '총 응답 수'를 연산하여 출력한다. (Req. 22)
